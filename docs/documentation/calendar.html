Calendar

since 0.3.0
DOM API No
JS API Yes
{% include notice.html %}

Methods

Method name Parameters Description
This plugin has no public methods.

Javascript options

Option name Required Default Type Description
element Yes undefined HTMLElement This is a reference to the container for the calendar or to an input that will act as the trigger.
date No new Date() Date The date the calendar will start at. Defaults to the current month.
navButtons No true boolean Should the nav buttons be shown.
format No yyyy/mm/dd string The format that the calendar should output when using an input field. yyyy, yy, mm, m, dd, d are all valid characters. Any separator can be used.
overlay No false boolean Should the calendar be shown as a modal rather than inline. Used when attached to an input.
